# Synthetic Data Sources

These files are generated by `scripts/generate_synthetic_data.py` using
Faker with a fixed seed. They simulate three real-world member data
sources a payer might reconcile: a legacy CSV system, a JSON partner
feed, and a CSV export from an API. The data is entirely synthetic —
no real PII — but mirrors the kinds of inconsistencies real integration
work involves: different column names, different date formats, variant
plan code vocabularies, typo-variant names, and leading-zero ZIP codes.

Total: 1,250 raw rows → ~600 unique persons after reconciliation.
Every run of the generator produces identical output (seed=42).
